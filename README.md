# Система обнаружения и сопровождения воздушных объектов

Прототип системы компьютерного зрения для охраны периметра режимных предприятий.  
Обнаруживает дроны, самолёты и вертолёты в реальном времени, сопровождает до 5 целей,
отображает рамки, координаты, таблицу и зону интереса.

---

## Модель

**Источник**: YOLOv8n (Ultralytics), дообученная на датасете AOD‑4v2
**Лицензия датасета**: CC BY 4.0  
**Классы**: `airplane`, `bird`, `drone`, `helicopter`  
**Размер**: ~6 МБ  

**Метрики после обучения (50 эпох, RTX 4060)**:

| Класс        | Precision | Recall | mAP50 | mAP50‑95 |
|--------------|-----------|--------|-------|----------|
| airplane     | 0.925     | 0.951  | 0.967 | 0.690    |
| bird         | 0.922     | 0.952  | 0.970 | 0.697    |
| drone        | 0.850     | 0.943  | 0.956 | 0.607    |
| helicopter   | 0.909     | 0.951  | 0.962 | 0.617    |
| **all**      | 0.819     | 0.811  | 0.830 | 0.540    |

Модель сохраняется в файле `best.pt`  
Основные классы (дрон, самолёт, вертолёт) детектируются с mAP50 > 0.95. Класс `bird` оставлен для фильтрации ложных тревог.

---

## Основные возможности

- **Детекция** – YOLOv8n с адаптивными порогами уверенности, CLAHE, резкостью, TTA (включается при необходимости).
- **Трекинг** – фильтр Калмана + Венгерский алгоритм (SORT), до 60 кадров удержания без обновлений, временнáя фильтрация уверенности и голосование по метке класса.
- **Интерактивная зона интереса** – выделение мышью (клавиша `Z`), динамическое отслеживание при движении камеры через оптический поток (Lucas‑Kanade).
- **Таблица сопровождения** – ID, тип, координаты, оценка угрозы.
- **Пауза видео** – кнопка «Пауза» останавливает поток для спокойного задания ROI.
- **Подавление ложных дубликатов** – удаление перекрывающихся боксов одинаковых классов.
- **Раздельные пороги классов** – drone (0.45), airplane (0.4), helicopter (0.35).

## Установка 
Нужны Python и Git. Для загрузки моделей нужен Git LFS.

```powershell
git clone https://github.com/rockettoom666-ops/Burevestnik.git
cd Burevestnik
git lfs install
git lfs pull
```

Создай виртуальное окружение и поставь библиотеки:

```powershell
py -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

Если PowerShell не дает включить окружение, один раз выполни:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\.venv\Scripts\Activate.ps1
```

### Запуск
Основная точка запуска:

```powershell
python .\код\прототип.py
```

После запуска:

1. Нажми `Работать с камерой`, чтобы выбрать веб-камеру.
2. Или нажми `Тест на видео`, чтобы открыть видеофайл.
3. Включи отслеживание в интерфейсе.
4. При появлении нового объекта программа покажет предупреждение и проиграет звук тревоги.

### Интерфейс и управление
- **«Работать с камерой»** – поиск и подключение веб-камеры (индексы 0–9).
- **«Тест на видео»** – открыть локальный видеофайл (mp4, avi, mov, mkv, webm).
- **«Пауза» / «Продолжить»** – приостановка потока (например, для выделения зоны).
- **«Остановить»** – полная остановка источника и сброс трекера.
- **«Показывать объекты»** – тумблер включения/отключения детекции.
«Задать зону (Z)» – включение режима выделения запретной области мышью прямо на кадре.
«Сбросить (R)» – удаление установленной зоны.
