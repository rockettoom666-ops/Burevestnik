# Буревестник

Основной запуск проекта: `burevestnik.py`.

Это оконное приложение на Python для обнаружения и сопровождения воздушных объектов на видеопотоке с камеры или из видеофайла. В интерфейсе есть список камер, загрузка видео для теста, рамки объектов, координаты центра, таблица целей, зона интереса, звуковое/визуальное оповещение и отладочный выбор модели.

## Что лежит в main

- `burevestnik.py` - основное приложение.
- `requirements.txt` - библиотеки Python.
- `модели/best.pt` - основная модель.
- `модели/best_1.pt` - вторая модель для сравнения через меню отладки.
- `ресурсы/тревога.wav` - звук тревоги.
- `презентация/` - материалы для защиты.

## Установка

Нужны Python и Git. Git LFS для текущих `best.pt` и `best_1.pt` не обязателен: эти две модели лежат в репозитории обычными файлами. Если потом будете добавлять более тяжелые модели, Git LFS лучше включить отдельно.

```powershell
git clone https://github.com/rockettoom666-ops/Burevestnik.git
cd Burevestnik
```

Создай виртуальное окружение и поставь библиотеки:

```powershell
py -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
python -m pip install -r requirements.txt
```

Если PowerShell не дает активировать окружение:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\.venv\Scripts\Activate.ps1
```

Проверка, что модели на месте:

```powershell
dir .\модели
```

В папке должны быть `best.pt` и `best_1.pt`.

## Запуск

Запускать нужно из корня проекта:

```powershell
.\.venv\Scripts\Activate.ps1
python .\burevestnik.py
```

После запуска можно выбрать камеру или загрузить видеофайл для теста.

## Если не запускается

Сначала проверь, что окружение активно и библиотеки поставлены:

```powershell
python -m pip install -r requirements.txt
python -B -m py_compile .\burevestnik.py
```

Если ошибка связана с моделью, проверь папку `модели/`. Приложение ищет модели именно там и показывает их в выпадающем списке отладки.

## Библиотеки

В `requirements.txt` лежат нужные прямые зависимости для `burevestnik.py`:

- `customtkinter` - оконный интерфейс.
- `opencv-python` - камера, видео и обработка кадров.
- `pillow` - вывод кадров в интерфейсе.
- `numpy` - работа с массивами и кадрами.
- `ultralytics` - YOLO-модели.
- `filterpy` - фильтр Калмана для сопровождения целей.
- `scipy` - сопоставление объектов между кадрами.

Зависимости, которые нужны самому `ultralytics` например `torch`, устанавливаются автоматически при установке `ultralytics`.

## Отладка

В левой панели есть блок отладки:

- выбор модели из папки `модели`;
- настройка шага анализа видео;
- показ фактического шага между анализами;
- показ времени последнего YOLO-анализа в миллисекундах.

Для камеры анализ идет каждый кадр. Для видео используется выбранный шаг, чтобы поток не превращался в слайд-шоу.
